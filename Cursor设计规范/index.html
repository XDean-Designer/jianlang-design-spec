0概述与 AI 使用指引

剑琅联盟 AI Ready v1.3 · 画布 390×844（高可延展）

本规范为剑琅联盟（含 RTP 商品零售模块）专属，基于 TDesign Flutter 与已落地主页面提取的 Token 定制。

AI 批量出图必读规则

强制约束：所有页面必须使用本规范中的 Design Token，禁止自行发明颜色、字号、圆角。组件命名对齐 TDesign 组件名（如 Navbar、Cell、TabBar）。画布宽 390px（固定）、高默认 844px（内容超出时可向下延展，宽不变），全局字体 PingFang SC，主要按钮圆角 4px，使用 4px 间距网格。
  1. 先选页面模式：从第 8 章选择列表页/详情页/表单页/工作台等模板。
  2. 再组合组件：从第 7 章按场景选用 TDesign 组件，保持 API 与视觉一致。
  3. 颜色只用 Token：主色 brand-6 (#F32F41)，文字/背景/边框从中性色和功能色中选取。
  4. 信息层级：标题 16–20px Medium，正文 14px Regular，辅助 12px，数据数字可用 20–28px Medium。
  5. B端气质：克制装饰、强调效率；列表优先于卡片堆叠；操作路径不超过 3 层。
  6. 输出格式：设计稿标注 Token 名称（如 --td-brand-color），而非裸 Hex 值。
  7. 品牌露出：启动页、登录页、Navbar 首页须使用官方 Logo（LOGO2.png）与应用名「剑琅联盟」。

1设计原则

高效 Efficient

减少操作步骤，常用功能 2 次点击可达。列表支持搜索、筛选、批量操作。表单支持自动填充与历史记录。

清晰 Clear

信息层级分明，关键数据突出。状态用颜色+文字双重标识。空状态、加载态、错误态均有明确指引。

一致 Consistent

全局统一导航、按钮、表单、反馈方式。同类业务使用相同页面结构与组件组合。

专业 Professional

视觉克制，避免过度圆角与渐变。品牌色用于关键操作与状态强调，不作为大面积背景。

视觉风格关键词

商务 · 简洁 · 可信赖 · 数据驱动 · 操作明确 · 留白适度 · 白底灰底交替分区

3.3 排版规则

  • 单行文字超出使用省略号 text-overflow: ellipsis
  • 多行描述最多 2 行，超出截断
  • 数字金额右对齐，使用千分位；日期格式 YYYY-MM-DD HH:mm
  • 中英文混排时英文与中文之间无需额外空格（中文产品惯例）
  • 禁止低于 10px 的可读文字（除 TabBar 图标下 10px 标签）

P-H01 剑琅首页（Figma 主页面 ★）

P-R01 RTP 选品上架（店主端）

结构：Navbar → 供货商条 → Tabs(全部/已上架) → Search+筛选 → ProductCard 双列网格
按钮：「上架到店」Primary 4px · 「已上架」Tag success

P-R02 商品详情（消费者/店主预览）

结构：大图 → 价格区(零售价+划线原价) → 规格行 → 优惠/配送 → 门店条 → 评价 → CartBar 固定底
底栏：店铺/客服/购物车(带 Badge) + 加购 Ghost + 购买 Primary

P-R03 价格编辑（供应商）

结构：Navbar → PriceRow 表格(原价/零售价/供货价/库存) → 分销渠道价折叠 → BottomBar 保存
规则：retail ≥ supply · 无上游零售指导价（PRD-214）

P-01 工作台首页（通用 B 端）

结构：Navbar → 数据概览卡 → 快捷入口 → 待办 Cell → TabBar
适用：非美业首页的通用工作台变体

P-02 列表页（通用）

结构：Navbar(返回+标题+筛选) → Search → DropdownMenu(可选) → 列表(Cell/SwipeCell) → 浮动 Fab 或 Bottom Bar「新建」
列表项：左标题+Tag 状态 · 右时间/金额 · 底部分割线
空状态：Empty + 「暂无数据」

P-03 详情页

结构：Navbar → 状态头(Tag+标题+元信息) → Collapse 分组(Cell 只读) → 时间线(Steps/自定义) → Bottom Bar(操作按钮)
状态头：brand-1 或 gray-1 背景 · 审批状态用大 Tag

P-04 表单页

结构：Navbar → Form(Cell 嵌 Input/Picker/Upload) → Bottom Bar「提交」
规则：必填 * 号 · 实时校验 · 提交后 Toast + 返回
分组：组标题 font-s text-secondary 上间距 24px 下间距 8px

P-05 审批流程页

结构：Navbar → 申请信息卡 → Steps 审批进度 → 审批意见列表 → Bottom Bar「通过」「驳回」
驳回：danger outline · 通过：primary · 需二次 Dialog 确认

P-06 登录 / 企业认证

结构：剑琅联盟 Logo 72px + 标题「剑琅联盟」font-xxl → Input 组 → primary block 按钮（圆角 4px）→ 协议链接 font-s
背景：white 或顶部 brand 渐变 · Logo 必须使用官方 LOGO2.png

P-07 消息中心

结构：Navbar → Tabs(全部/系统/审批) → 消息 Cell(未读 bold + 红点) → 下拉刷新
未读：标题 font-base Medium · 已读 text-secondary Regular

P-08 个人中心

结构：头部(Avatar 48 + 姓名 font-l + 部门 font-s) → Cell 组(设置/关于/退出) → TabBar
退出：Cell 文字 error-6，无箭头

9交互与反馈规范

9.1 动效

场景时长曲线
页面转场300msease-in-out
Popup/BottomSheet 弹出240mscubic-bezier(0.38, 0, 0.24, 1)
按钮点击100msopacity 0.7 或背景色变深
Toast 显隐200msfade
列表加载Skeleton shimmer 1.5s 循环—

9.2 反馈规则

信息提示 → Toast（轻量）或 Message（需手动关闭）
警告/待办 → NoticeBar 或 Dialog 确认
  • 操作成功：Toast「操作成功」2s 自动消失
  • 操作失败：Toast 或 Dialog 展示具体原因
  • 网络错误：Result 页 + 「重试」按钮
  • 删除/驳回：Dialog 二次确认，确认按钮 danger
  • 表单提交：按钮 Loading 态，防重复点击
  • 列表加载更多：底部 Loading + 「没有更多了」font-s text-placeholder

9.3 手势

  • 右滑/返回按钮：返回上一页
  • 下拉：PullDownRefresh 刷新列表
  • 左滑 Cell：露出快捷操作（SwipeCell）
  • 长按：批量选择模式（列表页）

10B端业务专项规范

10.1 状态色映射（通用）

业务状态Tag 类型颜色
草稿 / 未提交defaultgray-1 / text-secondary
待审批 / 进行中brandbrand-1 / brand-6
已通过 / 已完成successsuccess-1 / success-6
已驳回 / 已取消errorerror-1 / error-6
即将到期 / 待补充warningwarning-1 / warning-6
已归档 / 只读defaultgray-2 / gray-8

10.2 数据展示

  • 金额：¥ 前缀 + 千分位 + 2 位小数，font-m Medium，正数 text-primary，负数 error-6
  • 百分比：保留 1 位小数 + % 后缀
  • 日期：YYYY-MM-DD；时间：HH:mm；完整 YYYY-MM-DD HH:mm
  • ID/编号：font-s text-secondary，可复制，点击 Toast「已复制」
  • 大数字概览：font-data + 单位 font-s text-secondary

10.3 权限与空状态

  • 无权限：Result 页 + lock 图标 + 「暂无访问权限」+ 联系管理员
  • 无数据：Empty + 引导创建按钮
  • 搜索无结果：Empty + 「未找到相关内容」
  • 维护中：NoticeBar + 全页 Result

10.4 表单 B 端规则

  • 字段分组：基础信息 / 业务信息 / 附件
  • 长表单：分步 Steps 或锚点 Tabs
  • 只读模式：Cell 样式，值 text-secondary，无 Input 边框
  • 审批意见：Textarea 必填，最少 5 字
